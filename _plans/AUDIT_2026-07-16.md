# Omnitext audit, 2026-07-16

Four-track audit (architecture, security/privacy, robustness/tests, features).
Cross-checked against _plans/AUDIT_2026-07.md so items that already shipped are not re-listed.

## Verdict

Unusually good shape for the scope (~40 editors, ~110 formats, 64 MB of lazy chunks,
entry chunk ~444 KB). The core is a clean plugin kernel: src/core imports zero parsers
and zero editor widgets, resolution is fully data-driven, no ts-ignore, no any-casts
outside library boundaries. The privacy claim holds: no third-party runtime fetch, no
telemetry, no secrets committed, every untrusted-HTML path sanitized or sandboxed.
107/107 tests pass; CI gates Pages deploy and the APK on them.

Weaknesses concentrate in three places: silent failure on save, an untested
crash-recovery layer, and no large-file guard.

## Issues (ranked)

1. **Save fails silently when a serializer throws** (src/main.ts ~983-1035).
   getBytes/getText/encodeText run OUTSIDE the try/catch; both call sites
   (Save button main.ts:1386, Ctrl+S main.ts:1379) discard the rejection.
   No toast, dirty stays true, and autosave/snapshot (main.ts:668) throws the same
   way, so crash recovery also stops. Document becomes unsaveable with zero signal.
   Fix: wrap serialization, surface notify.saveFailed.
2. **IndexedDB persistence layer has zero tests** (src/core/session-store.ts:60-90,
   src/tools/version-store.ts:51-101). Only crash-recovery copy of unsaved edits.
   Quota-exceeded prune-and-retry logic untested; a regression that prunes the
   current snapshot ships green. Add fake-indexeddb tests: save->loadLatest round
   trip, quota branch keeps current session id, version-store add-then-cap never
   evicts the just-added snapshot.
3. **No size guard on open**: decodeBytes (src/core/encoding.ts:24-59) decodes the
   whole buffer to one string on the main thread; openBuffer (main.ts:718) has no
   threshold. Archive listing is fully synchronous too (core/archive.ts:29 unzipSync;
   core/libarchive.ts:43-48 reads every entry into memory). 500 MB file = frozen tab.
   Route huge files to hex/streamed handling.
4. **Eager-bundle leaks**:
   - formats/codemirror-formats.ts:1 statically imports StreamLanguage, dragging
     @codemirror/language (+state/lezer) into the entry chunk. Move the value import
     inside the already-async loadMode(); keep only the type import at top.
   - formats/blank-templates.ts:1 eagerly imports fflate (only needed in
     blankTemplate()). Make it a dynamic import.
5. **No CSP on index.html.** Sanitized HTML is injected into the top document
   (markdown/ipynb/rtf). Add a CSP meta tag (default-src 'self'; object-src 'none';
   base-uri 'none') and hash the inline theme script (index.html:4-10).
   Only hardening item from the security pass; the rest was clean:
   - html/eml render in sandbox="" iframes (email adds a strict CSP, emailviewer.impl.ts:164-168)
   - markdown/ipynb via DOMPurify; main.ts innerHTML uses escapeText
   - low/optional: epub iframe uses allow-same-origin (no scripts, keep the invariant);
     svgedit postMessage targetOrigin "*" could be location.origin; email HTML could
     also be DOMPurify'd belt-and-suspenders.
6. **main.ts is a 2,008-line god file.** Extraction seams already exist: command
   palette (1545-1665), settings dialog (1484-1544), New dialog (1134-1305),
   encoding pill (1388-1483), workspace/ui providers (1771-1911) -> src/ui/*.ts.
   Related: fallback editor id "codemirror" hardcoded in 4 degradation paths
   (main.ts:527,528,639,1296) instead of read from engine config; mountDoc
   (main.ts:475-655, 180 lines) and the live-editor cache (main.ts:552-579)
   deserve extraction (LiveEditorCache class) plus tests.
7. **37 format impls repeat identity parse/serialize boilerplate.** Add
   textFormatModule(languageFactory?) to formats/_util.ts; collapses 15-20 files
   (python/css/shell/sql/...) to one-liners.
8. Small stuff:
   - ajv ships but is never used (drop it or wire JSON Schema validation).
   - Corrupt .gz falls through to a hex dump of the compressed bytes (main.ts:758-767)
     with no explanation.
   - downloadBytes (main.ts:1044-1051) revokes the object URL synchronously after
     click(); defer the revoke (can cancel downloads on some browsers).
   - Encoding tests only cover UTF-8/16 + windows-1252; the picker also offers
     shift_jis/euc-jp/gbk/big5/windows-1251/koi8-r with no round-trip/lossy-flag test.
   - .tar/.tgz routing special-cased in the shell (main.ts:742), the one place
     format knowledge leaked upward (known, commented).

## What is genuinely clean (preserve)

- Descriptor/.impl lazy split 100% consistent; all heavy libs in dynamic chunks.
- mountDoc failure containment is layered and robust (format-load / parse /
  editor-load / mount fallbacks, main.ts:500-642): a throwing parser never blanks the UI.
- Resource hygiene in sampled editors (media, model3d, dxf, email, pptx, svg) is
  correct: URLs revoked, observers disconnected, listeners removed.
- Android: signing via CI env secrets only, INTERNET permission only, no cleartext,
  google-services.json absent (plugin inert).

## Nice to have

Quick wins (S, mostly already-bundled pieces):
- Print / save-as-PDF of the current view.
- Manual language switcher persisted in settings (i18n is auto-detect-only).
- share_target in manifest.webmanifest ("share to Omnitext" on Android).
- Wire ajv for JSON/JSON5 Schema validation.
- Status-bar encoding / line-ending / BOM indicator.

Bigger bets (best value/effort first):
- App-wide find (Ctrl+F) beyond CodeMirror: filter archive entries, sqlite tables, hex.
- Editable hex editor ("no file fails to edit"; binary view is read-only today).
- Log viewer with level/regex filtering and tailing.
- Compare-two-files diff (jsdiff already ships; history already renders diffs).
- Recent-files list on start screen (needs stored FS handles + permission preflight).
- mbox viewer (reuse the .eml renderer); binary plist decoder (plist npm already scouted).
- Multi-file / folder drop with a tab or file-tree strip (folder drop is currently an
  unhandled rejection).
- Optional at-rest passphrase encryption for the IndexedDB cache (opt-in).

Still deliberately deferred from the original roadmap: share-link collaboration
(Yjs/Trystero), export/conversion pipeline, multi-document tabs, Tauri build,
Android write-back to the opened content URI + multi-file intents.

Dead core API surface to either wire or prune: CommandDescriptor.keybinding/global,
contributeCommands, events formatDetected/editorChanged/selectionChanged, hooks
beforeClose/willChangeFormat, StorageProvider, applyRemote.
