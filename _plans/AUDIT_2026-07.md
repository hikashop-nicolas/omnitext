# Omnitext family audit (2026-07-02, cleaned 2026-07-09)

Scope: omnitext (app shell) plus its three first-party libraries consumed as git
dependencies: richdoc (.docx/.odt), pdfedit (.pdf), sheetedit (.xlsx/.ods).
Method: full source read of each repo, tests and typecheck run as baseline,
every finding verified in code. Third-party library internals (Quill, Milkdown,
Filerobot, svgedit, ...) are out of scope by decision.

This file only lists OPEN items. Everything already fixed lives in
AUDIT_2026-07_SHIPPED.md next to it, with commits. Line-number references
predate the fixes and the sheetedit module split, so treat them as approximate.

---

## Cross-cutting themes

- **Real-file round-trip corpora**: richdoc has one; sheetedit and pdfedit
  still test only synthetic fixtures.
- ~~**Main-thread zip work**~~: DONE (both directions). **Save**: richdoc
  (htmlToDocxAsync/htmlToOdtAsync), sheetedit (writeWorkbookAsync) and omnitext's
  archive codec (readArchiveAsync/writeArchiveAsync) compress on a fflate Web
  Worker. **Open**: createDocxEditorAsync / createOdtEditorAsync /
  createSheetEditorAsync inflate the container off-thread via unzipAsync and hand
  a pre-inflated map to the parser (docxToParts/odtToParts/readWorkbook accept an
  optional map); the omnitext editor impls call these. A .docx used to inflate
  three times per open (sniff + parse + fonts) all on the main thread; now once,
  off-thread. Note: the *inflate* is what moved off-thread; the DOM-bound XML
  parse + model build inherently stay on the main thread (no DOM in a worker), so
  a very large document can still block briefly on parse. That residual is the
  "Large-document performance" item below (cooperative parse / worker-side
  parsing), not the fflate item.

## 1. omnitext (app shell)

### Gaps / missing

- **Archive re-pack loses fidelity.** Zip rebuild recompresses everything and
  drops timestamps/permissions/comments (core/archive.ts); tar keeps only
  regular files (no dirs/symlinks/modes/mtimes) and lacks GNU/PAX long-name
  support (core/tar.ts).
- **saveIntoArchive never emits documentSaved**, so history "Saved" snapshots
  do not fire for archive-entry saves.
- **Single file only.** Drop/input take files[0]; dropping a folder is an
  unhandled rejection; no size guard before arrayBuffer().
- **.xls lossiness is invisible**: formulas/styles dropped and only the first
  sheet read, documented in a code comment only.
- **Adjacent types fall to hex**: .ppt/.odp, .doc, .xlsm/.docm, .svgz, .heic,
  .tiff, .odg. (.mkv and the other media containers route to the media viewer
  with a remux fallback since 2026-07-09.)
- **RTF view-only with no escape hatch** (no extract-text/convert action).
- **ajv is shipped but never used** (plan promised JSON schema validation).
  Use it or drop it.
- **Android save is always the share sheet**; no write-back to the opened
  content URI, cache temp files never cleaned, base64 bridge doubles memory.
  Intents handle one file (no ACTION_SEND_MULTIPLE).
- **i18n bypasses**: hardcoded English in table/tree/preview/archive/binary/
  image/rtf/latexpreview/filerobot impls; no manual language override;
  privacy.html English-only. (media impl localized 2026-07-09.)
- **Untested surface**: main.ts (open/save/archive/nav/dirty logic) has zero
  tests; no DOM tests at all (vitest env is node).
- **README staleness**: credits docxedit/odtedit instead of richdoc;
  binary-viewers.ts claims SVG is handled by the xml format when a dedicated
  svg format + svgedit exist.

### Improvements

- **Binary dirty flag lost on view switch**: changeEditor restores dirty state
  only for text docs, so switching image editor to viewer clears the unsaved
  dot and disarms beforeunload while edits are unsaved.
- **documentOpened emitted on mere view switches**, causing spurious "Opened"
  history snapshots.
- **Dead core API**: CommandDescriptor.keybinding/global, contributeCommands,
  applyRemote, StorageProvider, events formatDetected/editorChanged/
  selectionChanged, hooks beforeClose/willChangeFormat are declared and never
  referenced. Wire up or prune.
- **Two per-format-preference mechanisms, one dead**: engine preferredByFormat
  never populated; the app keeps its own localStorage map.
- **Table editor scalability/keyboard**: one input per cell, no virtualization,
  no Enter/arrow navigation, no sticky header. Now only serves .xls and the
  alternate Table view (csv/tsv default to the virtualized sheetedit); same
  class of issue for huge JSON in the tree view.
- **Settings offers A4/Letter only while New offers 9 sizes**, and
  saveSettingsDialog coerces anything else to a4.
- **Status "reason" pill leaks engine jargon** (native/view/fallback).
- **downloadBytes always uses application/octet-stream** even when the format
  MIME is known.
- **New-dialog format labels are raw ids** ("json5", "restructuredtext").
- **Toast UX**: one slot, fixed 5s, newer messages overwrite errors; errors
  should persist until dismissed.
- **Triple-duplicated write-out path** (saveFile/saveIntoArchive/exportFile);
  exportFile also swallows native save errors.
- **No permission preflight on stored FS handles** before createWritable; a
  revoked handle surfaces as a generic toast.
- **History panel**: no restore confirmation, no per-version delete/export UI,
  no auto-refresh while the doc changes.
- **Archive viewer**: flat alphabetical list, no folder grouping, no filter box.
- **i18n plural footgun**: t() returns the raw key for plural entries looked
  up without count; add a dev warning.
- **Shortcuts**: only Cmd/Ctrl+S and O are global; shift/alt combos are
  excluded wholesale so Cmd+Shift+S can never be Save As.
- **A11y leftovers (own code)**: New-format combobox lacks
  aria-activedescendant and option ids; table editor grid has no role=grid or
  header semantics.

### Feature ideas

- **Recent files / session list on start** (M); tabs/multi-document (L).
- **Export/conversion pipeline** (M): Markdown to HTML (marked already
  bundled), Markdown/HTML to PDF via print CSS, RTF to text; wasm-pandoc only
  if demand proves out (L).
- **Print / save-as-PDF of the current view** (S/M).
- **Subtitle editor for .srt/.vtt** (M): table with timecode validation and
  shift-all; unserved offline niche.
- **Archive editing** (M): add/delete/rename entries, extract-all, new zip.
- **Editable hex editor** (M): "no file fails to open" becomes "no file fails
  to edit".
- **JSON Schema validation via the already-shipped ajv** (S).
- **Diff/compare with a file on disk** (M): jsdiff already shipped, history
  panel already renders diffs.
- **Document stats + line-ending/encoding indicator** (S).
- **Language switcher + more locales** (S).
- **WebDAV storage module** (L): StorageProvider interface already defined;
  works client-side against Nextcloud-style CORS-friendly servers.
- **PWA share target** (S): "share to Omnitext" on the web app.
- **ffmpeg.wasm decode fallback for browser-less codecs** (L): WMV, MPEG-2,
  DivX-era files; ~30 MB lazy wasm, COOP/COEP headers Pages can't set
  (coi-serviceworker workaround vs the PWA SW), LGPL-build licensing care.
  Only if demand shows; container-level cases are already remuxed.
- **Real-time collaboration (Yjs + Trystero share links)** (L): on-plan,
  serverless fit, text editors first; gate on demand.
- **Tauri desktop build** (M/L): plan written and deferred; real file
  association and in-place saves on desktop.

## 2. richdoc

### Gaps / missing

- **w:sz/szCs pairs collapse to one size on save.**
- **List rendering fidelity**: the editor shows browser-default markers only
  (no roman/custom glyphs, no multilevel "1.1.1" labels in the editing
  surface), and ODT list styles are still re-pointed at a generic definition
  on save (the docx side now reuses numIds).
- **sniffFormat defaults any unknown bytes to docx**, so a corrupt file lands
  on the docx adapter before the read-only latch catches it.
- **Pre-existing PAGE/NUMPAGES/DATE fields are static passthrough**: a Word
  footer page number shows the same cached value on every page; a field
  inserted in richdoc degrades to passthrough after one save/reopen. Write
  side exists; this is a read-side gap.
- **ODT parity deltas**: headings capped at H3 (docx does H6), no page break
  capability (plumbing exists: data-odt-break-before), tracked deletions
  flattened to plain text, no tracked-formatting regions, comment
  replies/reactions absent, no live field recognition; ODF text:section
  elements (mid-page multi-column) are passthrough.
- **Per-paragraph RTL not rendered**: w:bidi now survives saves via the pPr
  stash, but the editor does not display per-paragraph direction.
- **Images**: no crop/rotate/borders; EMF/WMF placeholder-only (preserved).
- **Hyperlinks**: w:tooltip not modeled; every save mints new rels for every
  external link, orphaned rels accumulate per save.
- **Tables**: no cell vertical alignment, no cell shading UI, no header-row
  repeat, no table alignment/centering, no cell padding.
- **Comments**: bodies are plain-text only (no rich formatting in a comment).
- **Find and replace**: single-text-node matches only (misses matches split by
  a bold span), body only (not headers/footers/notes), no regex backreferences.
- **Suggest mode misses IME composition** (CJK input bypasses tracking) and
  paste is plain-text only.
- **Custom footnote number formats display as 1..n decimal in-editor.**
- **Document heading styles not rendered** (skipId excludes them from the
  injected CSS).
- **Tests**: find-replace, table-edit UI, suggest-mode typing (beforeinput),
  comments panel interactions, page-view rulers/zoom untested;
  no output validation (schema sanity of produced document.xml).

### Improvements

- **Large-document performance**: pagination measures layout after every block
  append (O(n) synchronous reflows); images ride as base64 in the live DOM and
  every cleanBody() string; the save zip and the read unzip are both off-thread
  now, but the DOM-bound parse/model build stays on the main thread.
- **Deduplicate adapters**: two near-identical part minters, rels-max-rId scan
  copy-pasted 6+ times; dedupe hyperlink rels by target.
- **API ergonomics for omnitext**: top-level EditorOptions out of sync with
  core (missing fileName/defaultPageSize/paginated/zoom); isDirty() never
  resets after getBytes(); missing onError, markClean(), focus(), getFormat(),
  per-instance locale.
- **Replace remaining window.prompt flows** (furigana, list start-at) with
  the dialog kit (i18n + a11y consistency).
- **A11y leftovers**: toolbar toggles lack aria-pressed; dropdown menus lack
  menu semantics/arrow keys/Escape; accept-reject and reaction popovers are
  mouse-only; find match counter not aria-live.
- **Touch**: image resize handle still mouse-only.
- **Field liveness on read**: map PAGE/NUMPAGES/DATE/... fldSimple and ODF
  field elements to the existing live spans (read-side only).
- **Cross-node find matching** via a per-block text index.

### Feature ideas

- **ODT page break** (S): write fo:break-before; plumbing exists.
- **Change navigation panel** (M): next/prev tracked change, per-author
  sidebar.
- **Blank document factories** createBlankDocx()/createBlankOdt() (S).
- **Clipboard-paste image insertion** (S; drag-drop already ships).
- **Search across headers/footers/notes** (S, after band indexing).
- **Ruler indent handles** (M): first-line/hanging indents in the UI.
- **List style picker** (M): roman/letters/custom bullet per level.
- **Cell vertical alignment + table alignment controls** (M).
- **Zoom presets and fit-page** (S). **Shortcut cheat-sheet dialog** (S).
- **Autosave hook** (S): debounced onChange bytes or periodic getBytes +
  markClean.
- **Page thumbnail navigator** (L). **Cross-format export docx <-> odt** (L).
- **Collaboration hooks** (L): operation events, prerequisite for shared
  editing.

## 3. pdfedit

### Gaps / missing

- **RTL/bidi absent**: item.dir ignored; heuristics assume LTR; new Arabic
  text would export unshaped.
- **Tategaki editing is partial**: vertical blocks never use the
  glyph-preserving path, so any edit redraws the whole block via Unicode
  re-encoding (vertical-variant glyphs become horizontal variants, column
  indents lost); detection is page-level all-or-nothing so mixed pages
  misgroup; ruby/furigana columns merge/split unpredictably.
- **Cover rectangle is a flat color patch**: visible on gradient/image
  backgrounds; justified glyph-preserve blocks left-align; glyph-preserve
  lines never wrap so long insertions overflow; drawRuns overflow overlaps
  content below.
- **Images**: cannot re-parent across pages after insert; existing PDF images
  cannot be selected/moved/removed; no paste/drag-drop.
- **Annotations absent** (highlight/note/ink; only new-link writing exists);
  existing links not clickable in-editor.
- **AcroForm fill/edit absent**; stale field values can sit under cover
  rectangles.
- **Page operations absent**: no rotate/reorder/delete/insert/merge.
- **Rotated pages (/Rotate 90/270) broken on export**: cover boxes and
  replacement text are axis-aligned in unrotated user space.
- **Cross-paragraph selection/copy impossible**; copying from
  glyph-unreliable fonts yields PUA garbage.
- **User-password PDFs**: the password prompt opens them, but editing an
  unlocked-by-password PDF is still unsupported.
- **Large PDFs**: rendered canvases are retained forever (no page.cleanup() /
  off-screen eviction) and multiple full byte copies are held; rendering
  itself is now lazy.
- **No recovery when the shared worker wedges**; a wedged worker takes down
  all subsequent documents in the same page.
- **Scanned PDFs give no "no editable text" feedback** in the UI.
- **Save is a full rewrite**: signatures invalidated, linearization lost,
  PDF/A can break; no incremental mode; no metadata-survival test. Structure
  tree/tagged PDF not maintained.
- **Tests**: the paragraph-reconstruction heuristics are unexported and
  untested; e2e uses a single-page standard-Helvetica fixture, so
  embedded-font reuse, glyph-preserve, PUA, vertical and multi-page paths have
  zero e2e coverage.

### Improvements

- **Split src/index.ts** (~2400 lines, huge closure): toolbar, image boxes,
  render pipeline, export, sampling helpers are natural seams; also makes the
  heuristics testable. (sheetedit got this treatment; pdfedit has not.)
- **One getImageData per page** instead of per-line/per-run probes: dominant
  cost on text-dense pages.
- **Off-screen canvas eviction** (rendering is lazy now; eviction is not).
- **Sharp zoom**: render at devicePixelRatio and debounce a re-render at the
  effective scale; CSS-only zoom is blurry at 200%+; debounce saveZoomPct
  localStorage writes.
- **Replace document.execCommand** for bold/italic/link with the existing
  Range-surgery applyStyle (unifies the two styling paths).
- **Content-stream tokenizer bugs**: literal-string escapes mishandled (octal
  \101 becomes "101", \n becomes "n"); anchor engine skips Form XObjects;
  visibility detection misses sc/scn/ExtGState alpha/OCG; simple fonts without
  Widths anchor at width 0.
- **Vertical: per-region detection + wire anchors into vertical blocks** so
  unchanged tategaki keeps original glyphs (machinery exists, not wired).
- **getState copies the full original per call**: heavy for a snapshotting
  host; share an immutable reference or document the cost.
- **A11y leftovers**: role=textbox overlays lack accessible names; no
  "Page N of M" labels (canvas aria-hidden); toolbar lacks roving tabindex;
  aria-pressed can go stale after execCommand without selection change.
- **Naming nit**: the dblclick handler shadows the i18n t import.

### Feature ideas

- **Ctrl/Cmd+wheel zoom** (S). **Whiteout tool** (S).
- **Paste/drag-drop images** (S).
- **Metadata editor** (S). **getText()/extraction API for hosts** (S).
- **Highlight/underline/sticky-note annotations** (M). **Freehand ink /
  signature drawing** (M).
- **Page operations sidebar** (L): thumbnails with delete/reorder/rotate/
  insert/merge.
- **AcroForm filling** (L). **True redaction** (L).
- **Existing-image select/move/delete** (L).
- **More locales (ja first)** (S). **Richer e2e fixtures** (embedded-font,
  multi-page, vertical) (S/M).

## 4. sheetedit

### Gaps / missing

- **Dates: display-side residue.** Formula cells don't inherit a date format
  from their inputs (=A1+1 on a date shows a serial until formatted), and the
  ods number-format picker maps formats to ODF value types without writing
  number:*-style data styles, so other apps render their default format for
  the type.
- **Formula limitations**: array formulas collapse to [0][0]; defined names
  not resolved in recalc.
- **Frozen panes not rendered; hidden rows/cols shown as normal.**
- **Phonetic strings**: rPh furigana runs are concatenated into cell text on
  read (Japanese workbooks read base text + furigana appended).
- **Tests**: nothing for array formulas, cross-sheet recalc, circular refs,
  REPEAT_CAP, phonetics; toolbar/border-popover UI paths untested.

### Improvements

- **Exclude rPh when joining t runs** in shared/inline strings.
- **destroy() leaks window pointerup/pointercancel listeners** (and any open
  border popover).
- **renderGrid full rebuild on every style/merge/resize** loses focus and
  scroll; patch affected tds instead. applyStyle stops silently after 4000
  cells mid-selection.
- **API ergonomics**: no markClean() so isDirty() can never reset after save;
  no programmatic cell access; setLocale unreachable (exports expose "." only).
- ~~**getBytes() is sync zipSync behind a Promise**~~: DONE. getBytes now uses
  writeWorkbookAsync (fflate async zip on a worker), so big workbooks do not jank.
- **Recalc dependency build is O(refs x formulaCells)** for ranges; bucket per
  sheet with a row index.
- **Select-all corner click leaves a stale anchor** for shift-click extends.
  Dead expression in typedValue.
- **Wrap and vertical alignment don't display in the grid** (single-line
  <input> cells); they persist to the file. Underline flavours (dotted, ...)
  degrade to solid when a styled cell is re-styled.
- **A11y leftovers**: border popover has no Escape/focus management; sheet
  tablist lacks arrow-key navigation.

### Feature ideas

- **Frozen panes rendering from the file's pane element** (M).
- **Defined-name support in recalc** (M).
- **options.locale + re-exported setLocale** (S). **More locales (ja)** (S).
