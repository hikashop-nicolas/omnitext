# Omnitext family audit (2026-07-02, updated 2026-07-03)

Scope: omnitext (app shell) plus its three first-party libraries consumed as git
dependencies: richdoc (.docx/.odt), pdfedit (.pdf), sheetedit (.xlsx/.ods).
Method: full source read of each repo, tests and typecheck run as baseline,
every finding verified in code (file:line references throughout). Third-party
library internals (Quill, Milkdown, Filerobot, svgedit, ...) are out of scope
by decision.

## Shipped (correctness sprint, 2026-07-03)

The data-loss/corruption shortlist below drove a first fix sprint. Done and
released (lib commit; omnitext bumped to APK build 190):

- sheetedit: Escape no longer commits the formatted display over a formula or
  value (was: one keypress turned "=B2*2" into the literal 6). Commit 78615c7.
- sheetedit: shared formulas resolve on read and de-share safely on edit; stale
  xl/calcChain.xml dropped and fullCalcOnLoad set when formulas change (was: an
  Excel "repair" trigger and stale cached values). Commit 78615c7. Also moved
  dist to build-on-install so consumers stop getting stale code (90981ea).
- pdfedit: owner-password PDFs save (ignoreEncryption on both pdf-lib loads);
  images insert on the page in view instead of always page 1. Commit c3e9ea6.
- richdoc: a document that fails to parse latches read-only with an error
  banner and getBytes returns the original bytes (was: a blank editable surface
  that could overwrite the real file). Commit 527f301.
- omnitext: UTF-16 files open as text (BOM check before the binary sniff);
  Open/New/drop/Back confirm before discarding unsaved edits; .gz files
  round-trip (re-gzip on save under the original name, top-level and archive
  entries). Commits 90c6f96, b711dff.

Trust sprint, first batch (2026-07-03):

- CI test workflows in all three libs: typecheck + vitest (+ cypress e2e for
  sheetedit/pdfedit) now run on every push/PR. First runs green.
- richdoc preserve-by-default serialization SHIPPED (commit ce7c855): the
  reader stashes unmodeled rPr/pPr content (data-docx-rpr / data-docx-ppr) and
  the writer merges it back at schema position, so small caps, character
  spacing, eastAsia fonts, underline flavours, keepNext/widowControl, outline
  levels, per-paragraph bidi, exact line spacing and first-line/hanging indents
  survive saves; list items keep their original numId (data-docx-numid), so
  roman/letter/custom numbering formats survive too, and numbering.xml stops
  growing. ODT parity for this mechanism is still open (its style-name model
  needs the equivalent stash).
- richdoc real-file round-trip corpus test: every demo/samples .docx now
  round-trips in the suite with text-fidelity assertions (corpus.test.ts).

## Top priorities remaining (cross-repo shortlist)

Data-loss and fidelity class, in rough order of user impact:

1. **sheetedit: ods saves regenerate every sheet's table body**, losing row
   visibility/default-cell-style attributes, header-row grouping, and
   covered-cell content, even on untouched sheets (src/index.ts:1498-1559).
2. **pdfedit: new characters the font cannot cover are silently deleted.**
   sanitizeStd strips everything outside Latin-1 + a few cp1252 extras
   (src/index.ts:467-477): typing Cyrillic/Greek/CJK into a Latin-font
   paragraph exports as missing text, no warning. Needs a Unicode fallback
   font (host-provided or lazy-loaded) or at minimum a warning.
3. **richdoc: ODT parity for preserve-by-default.** The docx side now merges
   stashed unmodeled properties back on save (ce7c855); the odt adapter still
   rebuilds styles from the modeled subset only (odt/write.ts:37-73), so the
   same class of loss remains there. Also still open on docx: in-cell
   paragraph formatting in tables (read.ts:321-323).
4. **omnitext: no crash recovery for binary documents.** Autosave and
   DocSnapshot are text-only (src/main.ts:461, core/session-store.ts:7-15); a
   crash mid PDF/DOCX/XLSX edit keeps only the 45s-debounced history snapshots.
5. **omnitext: new binary documents save as "untitled.txt"** and there is no
   rename/Save As, so a new DOCX/XLSX/PDF downloads binary bytes under a .txt
   name (src/main.ts:713, 966-975).

Cross-cutting themes (all four repos):

- ~~**No CI runs tests in any of the three libraries**~~ DONE 2026-07-03:
  test.yml (typecheck + vitest + cypress where present) runs on push/PR in all
  three libs.
- **Real-file round-trip corpora are the missing test layer.** richdoc now has
  one (corpus.test.ts over demo/samples); sheetedit and pdfedit still test
  only synthetic fixtures.
- **Undo/redo is missing or untrustworthy everywhere** outside plain-text
  editing: absent in sheetedit, absent for pdfedit toolbar/image ops, native
  contenteditable-only in richdoc where pagination reparenting invalidates it.
- **Errors are invisible.** Malformed/encrypted files end in console noise and
  a blank or wedged editor; none of the libs exposes an onError to the host.
  omnitext also does not wrap instance.mount() (src/main.ts:443).
- **Main-thread zip work**: fflate sync APIs are used for parse and save in
  richdoc, sheetedit, and omnitext's archive codec; big files jank or freeze
  the UI.
- **Theming**: richdoc and sheetedit hardcode their palettes; CSS custom
  properties would let omnitext theme them (and offer dark mode) cleanly.
- **Locales**: all four ship en/fr only; ja is the natural third given the
  tategaki/furigana investment.

## 1. omnitext (app shell)

### Gaps / missing

- **New binary documents save as "untitled.txt".** saveFile falls back to
  "untitled.txt" and there is no rename/Save As, so a new DOCX/XLSX/PDF
  downloads binary bytes under a .txt name (src/main.ts:713, 966-975).
- **No crash recovery for binary documents.** Autosave and DocSnapshot are
  text-only (src/main.ts:461, core/session-store.ts:7-15); a crash mid
  PDF/DOCX/XLSX edit keeps only the 45s-debounced history snapshots.
- **Legacy encodings corrupt silently.** Non-UTF-8/16 text decodes to U+FFFD
  and Save writes the corruption back; no warning, no encoding picker
  (core/encoding.ts:31-33).
- **Editor mount exceptions unhandled.** instance.mount() is not wrapped
  (src/main.ts:443); docx/odt adapters build inside a floating promise so an
  async construction error is an unhandled rejection (editors/docx.impl.ts:19-27).
- **Archive re-pack loses fidelity.** Zip rebuild recompresses everything and
  drops timestamps/permissions/comments (core/archive.ts:33-41); tar keeps
  only regular files (no dirs/symlinks/modes/mtimes) and lacks GNU/PAX
  long-name support (core/tar.ts:40-77).
- **saveIntoArchive never emits documentSaved**, so history "Saved" snapshots
  do not fire for archive-entry saves (src/main.ts:752-793 vs 736).
- **Unbounded IndexedDB growth.** SessionStore never deletes old sessions;
  VersionStore never prunes and stores full bytes per snapshot; quota errors
  are swallowed (core/session-store.ts, tools/version-store.ts).
- **Single file only.** Drop/input take files[0]; dropping a folder is an
  unhandled rejection; no size guard before arrayBuffer() (src/main.ts:646-683).
- **HTML default editor is destructive.** html.ts defaults to Quill; one
  keystroke replaces the document with getSemanticHTML() output (doctype,
  head, scripts, classes gone) (formats/html.ts:11, editors/quill.impl.ts:52-63).
  Default should be the text editor, Quill opt-in.
- **CSV: no delimiter sniffing.** Comma hardcoded (formats/csv/roundtrip.ts:31);
  semicolon CSVs (French Excel default) render as one column. detect() at 0.4
  for "line contains a comma" routes extensionless prose to the grid.
- **Table view cannot add/remove rows or columns** (ViewEdit is cell-only,
  core/types.ts:47-48).
- **.xls lossiness is invisible**: formulas/styles dropped and only the first
  sheet read, documented in a code comment only (formats/xlsx.impl.ts:10-12).
- **Adjacent types fall to hex**: .pptx/.ppt/.odp (no presentations at all),
  .doc, .xlsm/.docm, .epub, .svgz, .mkv, .heic, .tiff, .odg. .epub/.pptx are
  zips that could route to the archive viewer for free (formats/binary-viewers.ts).
- **RTF view-only with no escape hatch** (no extract-text/convert action).
- **ajv is shipped but never used** (plan promised JSON schema validation;
  json.impl.ts uses plain JSON.parse). Use it or drop it (package.json:53).
- **Not a PWA.** No service worker, no manifest: not installable, not offline,
  and stale pages 404 on lazy chunks after a redeploy (only editor chunks
  degrade gracefully, src/main.ts:372-384). The biggest misalignment with the
  local-first pitch.
- **Android save is always the share sheet**; no write-back to the opened
  content URI, cache temp files never cleaned, base64 bridge doubles memory
  (core/platform.ts:49-78). Intents handle one file (no ACTION_SEND_MULTIPLE).
- **i18n bypasses**: hardcoded English in table/tree/preview/archive/binary/
  image/media/rtf/latexpreview/filerobot impls (e.g. editors/tree.impl.ts:168,
  archive.impl.ts:69-70, binary.impl.ts:69); no manual language override;
  index.html:60 status default has no data-i18n; privacy.html English-only.
- **Untested surface**: main.ts (1450 lines of open/save/archive/nav/dirty
  logic) has zero tests; no DOM tests at all (vitest env is node). An openBuffer
  integration test would have caught the UTF-16 bug.
- **README staleness**: credits docxedit/odtedit instead of richdoc
  (README.md:36-39); binary-viewers.ts:7 claims SVG is handled by the xml
  format when a dedicated svg format + svgedit exist.

### Improvements

- **Binary dirty flag lost on view switch**: changeEditor restores dirty state
  only for text docs (src/main.ts:1026-1031), so switching image editor to
  viewer clears the unsaved dot and disarms beforeunload while edits are unsaved.
- **documentOpened emitted on mere view switches** (src/main.ts:474), causing
  spurious "Opened" history snapshots.
- **Dead core API**: CommandDescriptor.keybinding/global, contributeCommands,
  applyRemote, StorageProvider, events formatDetected/editorChanged/
  selectionChanged, hooks beforeClose/willChangeFormat are declared and never
  referenced (core/types.ts). Wire up or prune.
- **Two per-format-preference mechanisms, one dead**: engine preferredByFormat
  never populated; the app keeps its own localStorage map (core/resolve-editor.ts:72-76
  vs src/main.ts:156-171).
- **Table editor scalability/keyboard**: one input per cell, no virtualization
  (100k-row CSV freezes), no Enter/arrow navigation, no sticky header
  (editors/table.impl.ts:69-93). Same class of issue for huge JSON in tree view.
- **Theme sampled once at mount** (codemirror.impl.ts:29-32, milkdown.impl.ts:7-15);
  OS theme change mid-session leaves mismatched editors; no manual toggle.
- **Settings offers A4/Letter only while New offers 9 sizes**, and
  saveSettingsDialog coerces anything else to a4 (index.html:76-115,
  src/main.ts:1121-1128).
- **Status "reason" pill leaks engine jargon** (native/view/fallback,
  src/main.ts:468-470).
- **downloadBytes always uses application/octet-stream** even when the format
  MIME is known (src/main.ts:740-748).
- **New-dialog format labels are raw ids** ("json5", "restructuredtext")
  (src/main.ts:839).
- **Toast UX**: one slot, fixed 5s, newer messages overwrite errors
  (src/main.ts:1040-1048); errors should persist until dismissed.
- **Triple-duplicated write-out path** (saveFile/saveIntoArchive/exportFile,
  src/main.ts:714-731, 770-784, 1341-1345); exportFile also swallows native
  save errors.
- **No permission preflight on stored FS handles** before createWritable
  (src/main.ts:721-724); a revoked handle surfaces as a generic toast.
- **History panel**: no restore confirmation, no per-version delete/export, no
  pruning, no auto-refresh while the doc changes (tools/history.ts:192-219).
- **Archive viewer**: flat alphabetical list, no folder grouping, no filter box
  (editors/archive.impl.ts:57).
- **i18n plural footgun**: t() returns the raw key for plural entries looked up
  without count (i18n/index.ts:65-72); add a dev warning.
- **Shortcuts**: only Cmd/Ctrl+S and O are global; shift/alt combos are
  excluded wholesale so Cmd+Shift+S can never be Save As (src/main.ts:1057-1062).
- **A11y leftovers (own code)**: New-format combobox lacks aria-activedescendant
  and option ids (src/main.ts:856-899); table editor grid has no role=grid or
  header semantics.

### Feature ideas

- **Save As / rename** (S): fixes untitled.txt; prompt with the right extension.
- **PWA: manifest + service worker** (S/M): installable, offline, fixes
  stale-chunk 404s; biggest alignment win with the privacy pitch.
- **Command palette Cmd+K** (M): registry already exists; expose open/new/save/
  view/history/format actions.
- **Recent files / session list on start** (M); tabs/multi-document (L).
- **Encoding picker + legacy codepage decode** (M): "Reopen with encoding" via
  TextDecoder codecs, warn on lossy save; pairs with the UTF-16 fix.
- **Export/conversion pipeline** (M): Markdown to HTML (marked shipped),
  Markdown/HTML to PDF via print CSS, RTF to text; wasm-pandoc only if demand
  proves out (L).
- **Print / save-as-PDF of the current view** (S/M).
- **Subtitle editor for .srt/.vtt** (M): table with timecode validation and
  shift-all; unserved offline niche.
- **Archive editing** (M): add/delete/rename entries, extract-all, new zip.
- **Editable hex editor** (M): "no file fails to open" becomes "no file fails
  to edit".
- **JSON Schema validation via the already-shipped ajv** (S): detect $schema,
  surface in the lint gutter.
- **CSV delimiter/locale support** (S).
- **Diff/compare with a file on disk** (M): jsdiff already shipped, history
  panel already renders diffs.
- **Document stats + line-ending/encoding indicator** (S).
- **Language switcher + more locales** (S). **Theme toggle light/dark/system** (S).
- **EPUB viewer** (M): archive reader + sandboxed preview over the spine.
- **WebDAV storage module** (L): StorageProvider interface already defined;
  works client-side against Nextcloud-style CORS-friendly servers.
- **PWA share target** (S after PWA): "share to Omnitext" on the web app.
- **Presentations, read-only stopgap** (S): route .pptx/.odp to the archive
  viewer; full slide viewer later (L).
- **Real-time collaboration (Yjs + Trystero share links)** (L): on-plan
  (Phase 3), serverless fit, text editors first; gate on demand.
- **Tauri desktop build** (M/L): plan written and deferred; real file
  association and in-place saves on desktop.

## 2. richdoc

### Gaps / missing

- **Whole-body reserialization drops unmodeled properties document-wide**:
  SHIPPED for docx (commit ce7c855, stash + schema-position merge of rPr/pPr).
  Still open: the same class of loss on the ODT side (odt/write.ts styleFor
  rebuilds from the modeled subset), and w:sz/szCs pairs collapse to one size.
- **List numbering fidelity on save** SHIPPED for docx (original numId reused,
  definitions minted only for new lists, ce7c855). Still open: the editor
  renders browser-default markers only (no roman/custom glyphs, no multilevel
  "1.1.1" labels in the editing surface), and ODT list styles are still
  re-pointed at a generic definition.
- **docx table cells lose in-cell paragraph formatting on any save**: cell
  paragraphs render as bare divs without paragraphInfo, and rebuildTable
  re-serializes every cell from the DOM, dropping in-cell pPr and lists
  document-wide (docx/read.ts:321-323, write.ts:581-808). The ODT side does
  this correctly (odt/read.ts:754); docx is behind its own sibling.
- **Parse failure fallback** SHIPPED (read-only latch, commit 527f301). Still
  open: sniffFormat defaults any unknown bytes to docx (src/index.ts:17-26), so
  a corrupt file lands on the docx adapter before the latch catches it.
- **Pre-existing PAGE/NUMPAGES/DATE fields are static passthrough**: a Word
  footer page number shows the same cached value on every page; a field
  inserted in richdoc degrades to passthrough after one save/reopen
  (docx/read.ts:638-644, odt/read.ts:703-705). Write side exists; this is a
  read-side gap.
- **ODT parity deltas**: headings capped at H3 (docx does H6), no page break
  capability (plumbing exists: data-odt-break-before, odt/write.ts:126),
  tracked deletions flattened to plain text, no tracked-formatting regions,
  comment replies/reactions absent, no live field recognition; ODF
  text:section elements (mid-page multi-column) are passthrough.
- **Per-paragraph RTL absent and lossy** (section-level RTL works): w:bidi/
  w:rtl per paragraph neither read nor re-emitted; mixed LTR/Hebrew documents
  lose direction on save.
- **Images**: external-URL img (pasted from web) silently dropped on save
  (docx/write.ts:171-172); no crop/rotate/borders; EMF/WMF placeholder-only
  (preserved).
- **Hyperlinks**: w:tooltip not modeled; every save mints new rels for every
  external link, orphaned rels accumulate per save (docx/write.ts:28-39).
- **Tables**: no cell vertical alignment, no cell shading UI, no header-row
  repeat, no table alignment/centering, no cell padding.
- **Comments**: cannot edit an existing comment's text; bodies plain-text
  only; new-comment input is window.prompt (toolbar/index.ts:259).
- **Find and replace**: single-text-node matches only (misses matches split by
  a bold span), body only (not headers/footers/notes), no regex backreferences
  (find-replace.ts:5-6, 104-123).
- **Suggest mode misses IME composition** (CJK input bypasses tracking) and
  paste is plain-text only (track-changes.ts:153-162).
- **Custom footnote number formats display as 1..n decimal in-editor.**
- **Document heading styles not rendered** (skipId excludes them from
  injected CSS, docx/read.ts:1179).
- **Tests**: all 197 adapter tests are synthetic XML; 20 real samples in
  demo/samples are never loaded; no test asserts unmodeled rPr/pPr survival
  (they do not); no malformed-input tests; find-replace/table-edit UI/suggest
  typing/undo untested.

### Improvements

- **Preserve-by-default serialization** (the fix for the top-3 item): stash
  original w:rPr/w:pPr and ODF style names per run/paragraph like tables stash
  tblPr/tcPr, merge modeled edits onto them, or re-serialize only dirty blocks.
- **Reuse original list numbering**: carry numId/ilvl (docx) and list
  style-name (odt) on li and write back unchanged; mint definitions only for
  new lists.
- **Command-based undo stack**: native contenteditable undo is invalidated by
  pagination reparenting (editor.ts:628-1243) and programmatic mutations
  (table-edit, image resize, accept/reject); undo is currently untrustworthy.
- **Paste pipeline**: no paste handler exists; external HTML lands raw and
  serializes unpredictably; add a normalizer mapping clipboard HTML onto the
  editor vocabulary and inlining images to data URLs.
- **Large-document performance**: pagination measures layout after every
  block append (O(n) synchronous reflows, editor.ts:915-942); images ride as
  base64 in the live DOM and every cleanBody() string; unzip/zip are
  main-thread sync.
- **Deduplicate adapters**: two near-identical part minters, rels-max-rId scan
  copy-pasted 6+ times (docx/write.ts:975-1320); dedupe hyperlink rels by target.
- **API ergonomics for omnitext**: top-level EditorOptions out of sync with
  core (missing fileName/defaultPageSize/paginated/zoom); isDirty() never
  resets after getBytes(); missing onError, markClean(), focus(), getFormat(),
  per-instance locale.
- **Replace remaining window.prompt flows** (comment, furigana, list start-at)
  with the dialog kit (i18n + a11y consistency).
- **Error banner + read-only latch on parse failure**; expose onError.
- **CSS theming tokens** (--rdoc-*): palette is hardcoded; enable dark chrome.
- **A11y leftovers**: toolbar toggles lack aria-pressed (toolbar/index.ts:853-863);
  dropdown menus lack menu semantics/arrow keys/Escape; accept-reject and
  reaction popovers are mouse-only; find match counter not aria-live.
- **Touch**: image resize handle still mouse-only (feature/images.ts:60+).
- **Field liveness on read**: map PAGE/NUMPAGES/DATE/... fldSimple and ODF
  field elements to the existing live spans (read-side only).
- **Cross-node find matching** via a per-block text index.

### Feature ideas

- **Real-file round-trip test corpus** (M): load-edit-save-diff every
  demo/samples file against an allowlist of expected changes; highest-leverage
  guard for the lossy-save class.
- **ODT page break** (S): write fo:break-before; plumbing exists.
- **Word/character count in the bottom bar** (S).
- **Comment text editing** (M). **Change navigation panel** (M): next/prev
  tracked change, per-author sidebar.
- **Print / export to PDF via print stylesheet over the page cards** (M).
- **Blank document factories** createBlankDocx()/createBlankOdt() (S): lets
  omnitext offer New without shipping template binaries.
- **Drag-drop and clipboard image insertion** (S).
- **Search across headers/footers/notes** (S, after band indexing).
- **Ruler indent handles** (M): also forces modeling firstLine/hanging, fixing
  that lossy gap.
- **List style picker** (M): roman/letters/custom bullet per level.
- **Cell vertical alignment + table alignment controls** (M).
- **Zoom presets and fit-page** (S). **Shortcut cheat-sheet dialog** (S).
- **Autosave hook** (S): debounced onChange bytes or periodic getBytes +
  markClean, for omnitext crash recovery (pairs with the omnitext binary
  autosave gap).
- **Page thumbnail navigator** (L). **Cross-format export docx <-> odt** (L).
- **Collaboration hooks** (L): operation events, prerequisite for shared editing.

## 3. pdfedit

### Gaps / missing

- **Silent character loss for uncovered glyphs** (top list, item 3): no
  bundled/host-provided Unicode fallback font, no warning (src/index.ts:467-477).
- **Style toggles change typeface**: any bold/italic toggle breaks styleSame
  and drops to Helvetica/Times/Courier (src/index.ts:2133); faux bold is a
  double-strike (2357).
- **RTL/bidi absent**: item.dir ignored (src/index.ts:1703-1709); heuristics
  assume LTR; new Arabic text would export unshaped.
- **Tategaki editing is partial**: vertical blocks never use the
  glyph-preserving path (anchors: [], index.ts:1173-1175), so any edit
  redraws the whole block via Unicode re-encoding (vertical-variant glyphs
  become horizontal variants, column indents lost, vertical.ts:109-136);
  detection is page-level all-or-nothing (index.ts:1815-1819) so mixed pages
  misgroup; ruby/furigana columns merge/split unpredictably.
- **Cover rectangle is a flat color patch**: visible on gradient/image
  backgrounds (index.ts:2165-2167); justified glyph-preserve blocks
  left-align; glyph-preserve lines never wrap so long insertions overflow;
  drawRuns overflow overlaps content below (2339-2368).
- **Images: insert-only.** Insert-on-page-in-view SHIPPED (commit c3e9ea6).
  Still open: cannot re-parent across pages after insert; existing PDF images
  cannot be selected/moved/removed; no paste/drag-drop.
- **Annotations absent** (highlight/note/ink; only new-link writing exists,
  index.ts:2371-2390); existing links not clickable in-editor.
- **AcroForm fill/edit absent**; stale field values can sit under cover
  rectangles.
- **Page operations absent**: no rotate/reorder/delete/insert/merge.
- **Rotated pages (/Rotate 90/270) broken on export**: cover boxes and
  replacement text are axis-aligned in unrotated user space (index.ts:2165,
  2351-2358).
- **Search absent.** **Cross-paragraph selection/copy impossible**; copying
  from glyph-unreliable fonts yields PUA garbage (index.ts:754-762).
- **Encrypted PDFs**: owner-password save SHIPPED (ignoreEncryption, commit
  c3e9ea6). Still open: no password callback for user-password PDFs
  (index.ts:1659), which still fail to open.
- **Large PDFs fragile**: eager sequential render of all pages, canvases
  retained forever, no page.cleanup(), doc/loadingTask never destroyed in
  destroy() (index.ts:1674-2005, 2203-2214); multiple full byte copies held.
- **Worker lifecycle unmanaged**: one process-wide worker, no recovery when it
  wedges (the dev-server failure mode); a wedged worker takes down all
  subsequent documents (index.ts:686).
- **Scanned PDFs give no "no editable text" feedback** (README documents it,
  UI does not).
- **Undo/redo partial to absent**: toolbar styling via Range surgery is not on
  the browser undo stack; image/box/alignment ops have no undo at all.
- **Save is a full rewrite**: signatures invalidated, linearization lost,
  PDF/A can break; no incremental mode; no metadata-survival test
  (index.ts:2075, 2201). Structure tree/tagged PDF not maintained.
- **Tests**: unit tests cover only the three pure modules; the paragraph
  reconstruction heuristics are unexported and untested; e2e uses a
  single-page standard-Helvetica fixture, so embedded-font reuse,
  glyph-preserve, PUA, vertical, multi-page have zero e2e; CI never runs tests.

### Improvements

- **Split src/index.ts** (2391 lines, ~1700-line closure): toolbar, image
  boxes, render pipeline, export, sampling helpers are natural seams; also
  makes the heuristics testable.
- **Destroy pdf.js resources in destroy()** (loadingTask.destroy(),
  page.cleanup()): biggest lever on the wedging/memory issue.
- **One getImageData per page** instead of per-line/per-run probes
  (index.ts:1798-1812, 1909-1914): dominant cost on text-dense pages.
- **Lazy page rendering** (IntersectionObserver, drop off-screen canvases).
- **Sharp zoom**: render at devicePixelRatio and debounce a re-render at the
  effective scale; CSS-only zoom is blurry at 200%+ (index.ts:718, 1684-1685);
  debounce saveZoomPct localStorage writes (1478).
- **Replace document.execCommand** for bold/italic/link with the existing
  Range-surgery applyStyle (unifies code, enables undo stack).
- **Error surface**: onError option, visible message on render failure
  (index.ts:2009), ignoreEncryption + onPassword paths.
- **Content-stream tokenizer bugs**: literal-string escapes mishandled (octal

## 4. sheetedit

### Gaps / missing

- **Escape destroys formulas** SHIPPED (Escape now cancels, commit 78615c7).
- **Shared formulas: stale children + corruption on master edit** SHIPPED
  (resolve-on-read, safe de-share, calcChain drop, commit 78615c7).
- **ods saves regenerate every sheet's body** (top list, item 2): row
  visibility and default-cell-style lost, header-row grouping flattened,
  repeated rows expanded, covered-cell content dropped (src/index.ts:1242,
  1498-1559). REPEAT_CAP 1024 truncation means genuinely dropped content on
  ods save (955, 1199-1201).
- **Undo/redo entirely absent**: every commit is immediate and irreversible.
- **Copy/paste absent as a feature**: no clipboard handlers; pasting an Excel
  range dumps TSV into one cell.
- **Row/column insert/delete absent**: "+ Row"/"+ Col" only extend the grid;
  no shifting, no reference rewriting, no deletion (src/index.ts:2216-2223).
- **Keyboard navigation minimal**: Enter and (buggy) Escape only; no arrows,
  Ctrl+Home/End, F2, Delete-to-clear (2421-2431).
- **Dates partial**: display-only; typed dates stored as strings; t="d" cells
  misparse; ods date/currency cells degrade to string/float on edit
  (src/index.ts:494-497, 1252-1270, 1740-1770). Locale numbers ("3,5")
  rejected despite the fr UI (148-151).
- **Formula errors silent**: unsupported function or typo keeps the stale
  cached value with no error badge (1693-1699); unknown sheet name silently
  reads sheet 1 instead of #REF! (1609-1615); circular refs get one stale pass,
  no warning (1676-1679). Array formulas collapse to [0][0] and CSE attributes
  are dropped on rewrite (1584-1586, 573-584). Defined names not resolved
  (evaluation parser has no onVariable, 1681-1692).
- **Rendering caps silently hide data**: ROWS_CAP 5000 / COLS_CAP 256, no
  banner (1785-1788); no virtualization (250k inputs on a big sheet freezes);
  frozen panes not rendered; hidden rows/cols shown as normal (and ods
  visibility lost on save, 1524-1527).
